# Galaxy Explorer 🌌
演示页面
https://zjoooooo.github.io/galaxy-explorer/
<img width="1920" height="945" alt="Image_2026-07-04_141404_478" src="https://github.com/user-attachments/assets/ce6388f3-5854-4c13-9d6c-daccac89bf24" />

一个交互式、由代码程序化生成的**银河系**，完全在浏览器中用 WebGL（three.js）
运行。没有后端、无需构建、无需安装依赖、不需要 API 密钥——打开页面即可探索。

你看到的一切都是加载时由代码生成的。**没有任何星系图片素材**——旋臂、星云气体、
星团、远处的星系和超新星，全都是数学运算的结果。

**可自由用于任何用途**——个人、商业、学术与教学——遵循 [MIT 许可证](LICENSE)。


## ✨ 特性

- **旋涡星盘** — 约 85,000 个尘埃点分布在四条密度波旋臂上，暖色内核过渡到冷蓝色
  旋臂，还有明亮的恒星团块、HII 电离氢区和蓝色星团。
- **体积感星云** — 一个连续的 fBm + Worley 噪声着色器场（域扭曲旋涡、尘埃带、
  发光核心），用气体包裹旋臂。
- **星系核球** — 平滑的多尺度发光核心，缓慢地“呼吸”。
- **深空背景** — 闪烁的星场，加上约 170 个遥远的小星系（椭圆星系、旋涡星系、
  侧向盘状星系、不规则星系）、一个致密星系团，以及一个类似仙女座的大邻居星系
  ——全部仅用两次绘制调用完成。
- **基于物理的环境事件** — 偶尔出现的流星、流星雨，以及遵循真实光变曲线
  （t².³ 上升 → 圆润峰值 → 双斜率衰减）的超新星，并带有四芒衍射星芒闪光。
- **电影级观感** — 选择性泛光（bloom）+ ACES 色调映射。
- **性能自适应** — FPS 监视器会在性能较弱的 GPU 上自动降低尘埃密度，让它在
  笔记本、电视和一体机上都能流畅运行。

## 🎯 适用场景

- **学习与教学** — 一个紧凑、易读的单文件示例，涵盖 WebGL、three.js、GLSL
  着色器、程序化生成、粒子系统和噪声（fBm/Worley）。非常适合天文、计算机图形学
  和创意编程课程。每个子系统都自成一体并有注释。
- **背景与屏保** — 用作落地页主视觉、桌面/副屏背景，或待机画面的氛围星系。
- **数字标牌与装置** — 大堂屏幕、博物馆/天文馆展示、活动背景板。可全屏运行
  （按 `F` 进入影院模式）。
- **一个起点** — fork 它，构建你自己的太空可视化、以星系为画布的数据可视化、
  游戏背景或艺术项目。

## 🚀 安装与运行

**无需安装。** 只要拿到文件，并通过 HTTP 提供服务即可（浏览器会在 `file://`
协议下屏蔽音频、纹理等部分功能）。

```sh
# 1. 获取代码
git clone https://github.com/zjoooooo/galaxy-explorer.git
cd galaxy-explorer

# 2. 用任意静态服务器托管，例如：
python -m http.server 8000
#   或：  npx serve .
#   或：  php -S localhost:8000

# 3. 打开 http://localhost:8000/
```

**部署到任意静态托管：** GitHub Pages、Netlify、Vercel、Cloudflare Pages、S3、
nginx 或任何 Web 主机。上传文件夹即可——没有任何需要构建的东西。

## 🎮 操作

| 操作 | 效果 |
|---|---|
| **拖拽** | 环绕相机（orbit） |
| **滚轮** | 放大 / 缩小 |
| **`F`** | 影院模式——隐藏所有 UI |
| **`M`** | 背景音乐 开 / 静音（可选，见下文） |

**URL 参数**

- `?fx=1` — 演示模式：每 4 秒强制触发一次流星 / 流星雨 / 超新星，让你立刻看到
  环境特效。
- `?tune=1` — 星云辉光着色器的实时滑块；实时调整观感，满意后把数值复制进
  tunables。

## 🎛️ 自定义

打开 `index.html`，编辑 `<script>` 顶部附近的 **tunables（可调参数）** 块：

```js
var BLOOM = { strength: 0.08, radius: 0.32, threshold: 0.42 }; // 辉光
var DUST  = { count: 85000, radius: 360, arms: 4, tight: 0.85, brightness: 0.5, ... };
var GALAXY_SPIN = 0.013; // 旋转速度（弧度/秒）
```

- `DUST.count` — 恒星/尘埃总量（硬件较弱时调低）。
- `DUST.arms` / `DUST.tight` — 旋臂数量，以及缠绕的紧密程度。
- `BLOOM.strength` — 整体辉光（0 = 关闭）。
- `GALAXY_SPIN` — 整个星系旋转的速度。

## 🎵 背景音乐

`audio/` 里自带三首环境音乐，并会**在打开页面时自动播放**（若浏览器拦截自动
播放，会在首次交互时启动）。用扬声器按钮（右下角）或按 `M` 静音 / 取消静音；
悬停按钮可按名称选择曲目。曲目会循环播放。

想更换音乐，把你自己的 `.mp3` 文件放进 `audio/`，并在 `index.html` 的
`MUSIC_TRACKS` 中列出文件名：

```js
var MUSIC_TRACKS = ['ambient 1.mp3', 'ambient 2.mp3'];
```

把 `MUSIC_TRACKS` 留空即可完全隐藏扬声器按钮。
**请不要提交你没有分享权利、受版权保护的音频。**

## 🛠️ 工作原理（简述）

原生 JavaScript + three.js r128（打包在 `vendor/` 下）。一个自包含的
`index.html`。星系在加载时构建一次：

1. 尘埃点散布在密度波旋臂上（一个 `Points` 点云）。
2. 星盘平面上的一个平面运行片元着色器，计算由旋涡调制的 fBm + Worley 云噪声，
   生成星云气体。
3. 精灵辉光构成核球；一个闪烁的远景球面作为背景天空。
4. 程序化纹理图集 + 面向相机的四边形批次绘制远处的星系。
5. 后期处理：`UnrealBloomPass` 产生辉光，再经过一次 ACES 色调映射。

环境特效（流星、超新星）是短生命周期对象，按计划生成并自行销毁——空闲时每帧
没有任何持续开销。
