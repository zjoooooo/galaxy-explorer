# Galaxy Explorer 🌌
デモページ
https://zjoooooo.github.io/galaxy-explorer/
<img width="1920" height="945" alt="Image_2026-07-04_141404_478" src="https://github.com/user-attachments/assets/ce6388f3-5854-4c13-9d6c-daccac89bf24" />

WebGL（three.js）だけでブラウザ上で完結する、インタラクティブなプロシージャル生成の**天の川銀河**です。バックエンドもビルド工程も、依存パッケージのインストールも API キーも不要 — ページを開くだけで探索できます。

画面に映るものはすべて、読み込み時にコードで生成されます。銀河の**画像アセットは一切ありません** — 渦状腕も星雲ガスも星団も、遠方銀河も超新星も、すべて数式です。

[MIT ライセンス](LICENSE)のもと、個人・商用・学術・教育を問わず**あらゆる用途で自由に利用できます**。


## ✨ 特徴

- **渦状円盤** — 4 本の密度波の腕に散りばめられた約 85,000 個のダストポイント。暖色のコアから冷たい青の腕へと色が移ろい、明るい星のノット、HII 領域、青い星団を伴います。
- **ボリューメトリックに見える星雲** — 連続的な fBm + Worley ノイズのシェーダーフィールド（ドメインワープした渦巻き、ダストレーン、輝くコア）が腕をガスで包み込みます。
- **銀河バルジ** — ゆっくりと「呼吸」する、なめらかなマルチスケールのグローコア。
- **ディープフィールドの星空** — またたく星野に加えて、約 170 個の小さな遠方銀河（楕円銀河、渦巻銀河、エッジオンの円盤銀河、不規則銀河）、豊かな銀河団、そしてアンドロメダ風の大きな隣人がひとつ — これらすべてをわずか 2 回のドローコールで描画します。
- **物理モデルに基づくアンビエントイベント** — ときおり流れ星や流星群、超新星が発生します。超新星は実際の光度曲線（t².³ で立ち上がり → 丸みを帯びたピーク → 2 段階の減光）に従い、4 本の回折スパイクの閃光を放ちます。
- **インタラクティブなビーコン** — クリック可能な 15 個のマーカー（太陽系、いて座 A*、有名な星雲、星団、恒星、ブラックホール）。クリックすると詳細な情報カードが開き、選択した天体には光る 3D リングが表示されます。
- **太陽系ガイド** — **太陽系**ビーコンから**太陽系を探索する**へ進み、太陽、8 惑星、冥王星、ケレス、小惑星帯、代表的な衛星を自由に周回・ズームできます。天体を選ぶとカメラを動かしたり太陽系を止めたりせず、すぐにイラスト付きの情報カードが開きます。地球カードからは**地球の空を探索する**を選べます。観察しやすいよう公転と自転は意図的にゆっくりにしていますが、相対的な方向と周期は視覚的な目安として保たれています。表示縮尺と初期位置は読みやすさのためのもので、実寸やリアルタイムの位置ではありません。
- **黄道十二星座の空** — 太陽系内の地球カードから、地球中心で回転できる星空ドームを開けます。106 個の実在の恒星（J2000 座標）から描かれた黄道十二星座には、それぞれ専用のカードが付いています。
- **5 言語対応** — ラベル、カード、ナビゲーションは英語・簡体字中国語・繁体字中国語・日本語・韓国語に切り替えられます（右上のセレクターで選択。設定は記憶されます）。
- **ローカルテクスチャ詳細** — 太陽系の天体は、フォーカス、ビューポート、端末性能に応じてローカルのテクスチャ階層を切り替えます。実行時のテクスチャサービスは不要です。
- **シネマティックな画づくり** — 選択的な bloom + ACES トーンマッピング。
- **パフォーマンスへの配慮** — FPS ウォッチドッグが非力な GPU ではダスト密度を自動的に下げるため、ノート PC やテレビ、キオスク端末でも滑らかに動作します。

## 🚀 インストールと実行

**インストールは不要です。** ファイルを取得して HTTP で配信するだけです（`file://` プロトコルでは、オーディオやテクスチャなど一部の機能がブラウザによってブロックされます）。

```sh
# 1. コードを取得
git clone https://github.com/zjoooooo/galaxy-explorer.git
cd galaxy-explorer

# 2. 任意の静的サーバーで配信する。例:
python -m http.server 8000
#   または:  npx serve .
#   または:  php -S localhost:8000

# 3. http://localhost:8000/ を開く
```

**静的ホスティングならどこにでもデプロイ可能:** GitHub Pages、Netlify、Vercel、Cloudflare Pages、S3、nginx、その他任意の Web ホスト。フォルダーをアップロードするだけで、ビルドするものは何もありません。

## 🎮 操作方法

| 操作 | 効果 |
|---|---|
| **ドラッグ** | カメラを周回させる |
| **スクロール / ピンチ** | ズームイン / ズームアウト |
| **ビーコンまたは天体をクリック** | 情報カードを開く。太陽系の天体を選んでもカメラと運動は変わらない |
| **戻る** | 星空 → 地球 → 太陽系の全景 → 銀河の順に 1 段階戻る |
| **`Esc`** | 段階的に戻る：カードを閉じ、現在の太陽系レベルを離れ、最後に太陽系を離れる |
| **`F`** | シネマモード — すべての UI を非表示 |
| **`M`** | BGM のオン / ミュート（任意。下記参照） |

**URL パラメーター**

- `?fx=1` — デモモード: 流れ星・流星群・超新星を 4 秒ごとに強制的に発生させ、アンビエントエフェクトをすぐに確認できます。
- `?tune=1` — 星雲グローシェーダーのライブスライダーを表示。見た目をリアルタイムに調整し、気に入った値を tunables にコピーできます。

## 🎛️ カスタマイズ

`index.html` を開き、`<script>` の冒頭付近にある **tunables** ブロックを編集します。

```js
var BLOOM = { strength: 0.08, radius: 0.32, threshold: 0.42 }; // グロー
var DUST  = { count: 85000, radius: 360, arms: 4, tight: 0.85, brightness: 0.5, ... };
var GALAXY_SPIN = 0.013; // 回転速度（ラジアン/秒）
```

- `DUST.count` — 星・ダストの総数（非力なハードウェアでは下げてください）。
- `DUST.arms` / `DUST.tight` — 渦状腕の本数と巻きの強さ。
- `BLOOM.strength` — 全体のグロー（0 でオフ）。
- `GALAXY_SPIN` — 銀河全体の回転速度。

## 🎵 BGM

`audio/` に 3 曲のアンビエントトラックが同梱されており、**読み込み時に自動再生されます**（自動再生をブロックするブラウザ向けに、最初の操作をきっかけに再生するフォールバック付き）。右下のスピーカーボタンか `M` キーでミュート / 解除でき、ボタンにホバーすると曲名でトラックを選べます。トラックはループ再生されます。

音楽を変更するには、自分の `.mp3` ファイルを `audio/` に置き、`index.html` 内の `MUSIC_TRACKS` にファイル名を列挙します。

```js
var MUSIC_TRACKS = ['ambient 1.mp3', 'ambient 2.mp3'];
```

`MUSIC_TRACKS` を空にすると、スピーカーボタン自体が非表示になります。
**共有する権利を持たない著作権付き音源はコミットしないでください。**

## 🛠️ 仕組み（概要）

Vanilla JavaScript + three.js r128（`vendor/` 以下に同梱）。自己完結した 1 つの `index.html` だけで構成されています。銀河は読み込み時に一度だけ構築されます。

1. 密度波の渦状腕に沿ってダストポイントを散布します（`Points` のクラウド）。
2. 円盤面に置いた平面上でフラグメントシェーダーを走らせ、渦巻きで変調した fBm + Worley の雲ノイズを計算して星雲ガスを描きます。
3. スプライトのグローがバルジを形作り、またたく遠方の球体が背景の星空になります。
4. プロシージャルなテクスチャアトラス + ビルボード化したクアッドのバッチで遠方銀河を描画します。
5. ポストプロセス: `UnrealBloomPass` でグローをかけ、その後 ACES トーンマップのパスを適用します。

アンビエントエフェクト（流星や超新星）は、スケジュールに従って生成されては自ら破棄される短命のオブジェクトです — アイドル時のフレームあたりの定常コストはゼロです。
